Manual de
Operaciones

Guía completa para gestionar, actualizar y publicar el sitio web de Taller Torke. Sin conocimientos de programación necesarios para las tareas cotidianas.

01 — ESTRUCTURA

Estructura de Archivos

El sitio está compuesto por archivos estáticos. Esta es la estructura completa que tenés que subir a Netlify:

📁 torke-site/ ← Esta carpeta es la que subís a Netlify │ ├── index.html ← Página principal (tienda, recursos, etc.) ├── fluidos.html ← Módulo FluidDB (fluidos y capacidades) ├── _redirects ← Configuración de URLs para Netlify ├── MANUAL.html ← Este archivo (no se sube, es local) │ └── 📁 img/ ← Todas las imágenes del sitio ├── 📁 hero/ ← Fotos del slideshow de portada │ ├── hero-1.jpg │ ├── hero-2.jpg │ └── hero-3.jpg ├── 📁 productos/ ← Fotos del catálogo de productos │ ├── castrol-5w30.jpg │ └── filtro-toyota.jpg ├── 📁 noticias/ ← Miniaturas del blog │ └── aceite-diesel.jpg └── taller.jpg ← Foto sección "Sobre nosotros"
ℹ️
¿Por qué se llama index.html?
Cuando alguien escribe torke.com en el navegador, el servidor busca automáticamente un archivo llamado index.html. Si no existe, da error. Por eso el archivo principal siempre debe llamarse así.
02 — PUBLICACIÓN

Publicar en Netlify

Netlify es la plataforma donde va a vivir el sitio. Es gratuita para sitios estáticos como el de Torke.

1
Crear cuenta en Netlify
Entrá a netlify.com → Sign up → podés registrarte con tu email de Gmail. No requiere tarjeta de crédito para empezar.
2
Subir el sitio
En el dashboard de Netlify, buscá la opción "Add new site" → "Deploy manually". Aparece una zona de drag & drop. Arrastrá la carpeta torke-site completa ahí. En segundos el sitio queda publicado en una URL temporal como random-name.netlify.app.
3
Comprar el dominio
Desde el panel de tu sitio en Netlify → Domain management → Add a domain → Buy a new domain. Escribís torke.com, si está disponible lo comprás con tarjeta. Cuesta aprox. $15 USD/año. Netlify configura automáticamente el HTTPS.
4
¡Listo!
En 5–10 minutos torke.com apunta a tu sitio con HTTPS activo. No necesitás tocar ninguna configuración adicional.

Actualizar el sitio después de cambios

Cada vez que modifiques archivos (productos, imágenes, textos), necesitás volver a subir la carpeta a Netlify:

1
Entrás a tu sitio en Netlify
app.netlify.com → tu sitio → pestaña Deploys.
2
Drag & drop de la carpeta actualizada
En la zona de deploys hay un área que dice "Drag and drop your site output folder here". Arrastrás la carpeta torke-site completa nuevamente.
3
Deploy automático
Netlify detecta los cambios y actualiza el sitio en segundos. torke.com muestra la versión nueva sin tiempo de inactividad.
03 — TIENDA

Agregar Productos al Catálogo

Todos los productos se gestionan en el archivo index.html. Abrís el archivo con cualquier editor de texto (Notepad++, VS Code, o hasta el Bloc de notas de Windows).

💡
Editor recomendado: Notepad++
Es gratis, en español, y resalta el código con colores. Descargalo en notepad-plus-plus.org. Hacé Ctrl+F para buscar texto dentro del archivo.

Cómo encontrar la lista de productos

Abrís index.html con tu editor y buscás (Ctrl+F) el texto:

const PRODUCTS = [

Ahí empieza el array con todos los productos. Cada producto tiene esta estructura:

Estructura de un producto{
  id: 1,                           // ← Número único, no repetir
  name: 'Castrol GTX 10W-40 4L',  // ← Nombre del producto
  cat: 'lubricantes',              // ← Categoría (ver lista abajo)
  desc: 'Descripción corta...',    // ← Texto que aparece en la tarjeta
  price: 120000,                   // ← Precio en guaraníes (sin puntos)
  oldPrice: null,                  // ← Precio anterior (para mostrar tachado)
                                   //   null = no mostrar precio anterior
  emoji: '🛢️',                     // ← Emoji que aparece si no hay foto
  img: '',                         // ← Ruta de la imagen (ver sección fotos)
  badge: 'hot',                    // ← 'new' | 'sale' | 'hot' | ''
},

Agregar un producto nuevo

Copiás el bloque de arriba, lo pegás ANTES del corchete de cierre ]; del array, y completás los datos. Ejemplo:

  // ← El último producto ya existente termina aquí con ,
  {
    id: 15,
    name: 'Aceite Repsol 5W-40 4L',
    cat: 'lubricantes',
    desc: 'Aceite sintético de alto rendimiento. API SN/CF. Para motores exigentes.',
    price: 210000,
    oldPrice: 240000,
    emoji: '🛢️',
    img: 'img/productos/repsol-5w40.jpg',
    badge: 'sale',
  },
];
⚠️
Importante: IDs únicos
Cada producto debe tener un id diferente. Si repetís un número, el carrito puede tener errores. Simplemente seguí la numeración: si el último es 14, el nuevo es 15.

Eliminar un producto

Buscás el bloque del producto (por su nombre o ID) y borrás todo desde la llave de apertura { hasta la llave de cierre }, inclusive. Guardás y listo.

Agregar una categoría nueva

Si necesitás una categoría que no existe (por ejemplo "Refrigerantes"), hay que hacer dos cambios en index.html:

1. Buscá (Ctrl+F) el texto class="cat-bar" y agregás un botón nuevo dentro:

<button class="cat-btn" onclick="filterCat('refrigerantes')">🧊 Refrigerantes</button>

2. En los productos que correspondan, usás cat: 'refrigerantes'.

06 — WHATSAPP

Número de WhatsApp

El número de WhatsApp aparece en varios lugares del sitio. Para cambiarlo, en index.html buscás (Ctrl+F):

595981000000

Reemplazás todas las ocurrencias (Ctrl+H en Notepad++) con tu número real, en formato internacional sin + ni espacios:

FormatoEjemplo
Con código de Paraguay595981234567
Sin guiones ni espacios595981234567 ✅
Con + o espacios+595 981 234 567 ❌

El número también aparece en la función orderWhatsApp(). Buscás:

const WA_NUMBER = '595981000000';

Y reemplazás con tu número.

Mensaje de WhatsApp al finalizar pedido

Cuando el cliente hace clic en "Finalizar pedido", se genera automáticamente un mensaje así:

🛒 *Pedido desde Torke Shop*

• Castrol GTX 10W-40 4L x2 — ₲ 240.000
• Filtro de Aceite Toyota OEM x1 — ₲ 38.000

*Total estimado: ₲ 278.000*

Por favor confirmar disponibilidad y forma de entrega. ¡Gracias!

Podés editar el texto del mensaje buscando en el JS la función orderWhatsApp().

07 — IMÁGENES HERO

Fotos de Portada (Slideshow)

El slideshow de la página principal tiene 3 slides. Cada uno tiene su propia foto de fondo.

🖼️
Slide 1
Archivo: img/hero/hero-1.jpg Tamaño: 900 × 700 px mínimo Formato: JPG o WebP Sugerencia: Foto del taller o de productos lubricantes
🖼️
Slide 2
Archivo: img/hero/hero-2.jpg Tamaño: 900 × 700 px mínimo Formato: JPG o WebP Sugerencia: Foto de filtros o herramientas
🖼️
Slide 3
Archivo: img/hero/hero-3.jpg Tamaño: 900 × 700 px mínimo Formato: JPG o WebP Sugerencia: Foto de repuestos o autos
1
Preparar la foto
La foto puede ser tomada con el celular. Idealmente horizontal (landscape). Evitá fotos muy oscuras, el texto encima necesita contraste.
2
Guardar con el nombre correcto
Renombrás la foto como hero-1.jpg y la copiás a la carpeta img/hero/. Reemplazás el archivo placeholder que ya estaba.
3
Descomentar la línea en el CSS
En index.html buscás /* IMG-HERO-1 */ y en la línea de abajo eliminás el /* y */ que rodean la línea background-image:url('img/hero/hero-1.jpg');

Antes del cambio:

/* background-image:url('img/hero/hero-1.jpg'); */

Después del cambio:

background-image:url('img/hero/hero-1.jpg');
08 — IMÁGENES PRODUCTOS

Fotos de Productos

📸
Especificación recomendada
Tamaño: 600 × 600 px (cuadrada). Formato: JPG o WebP. Fondo: blanco o neutro. Peso máximo: 200 KB por imagen.
1
Fotografiar o descargar la imagen
Podés fotografiar el producto sobre una superficie limpia, o usar la foto del proveedor/fabricante. Procurá que el producto se vea claro y centrado.
2
Nombrar el archivo
Usá nombres descriptivos sin espacios ni caracteres especiales. Ejemplos: castrol-5w30-4l.jpg, filtro-toyota-90915.jpg, bateria-60ah.jpg
3
Copiar a la carpeta
Guardás la imagen en torke-site/img/productos/
4
Vincular al producto en el JS
En el objeto del producto en index.html, completás el campo img:
img: 'img/productos/castrol-5w30-4l.jpg',
🌐
También podés usar URLs externas
Si tenés las fotos en Google Drive, Dropbox o cualquier CDN, podés pegar la URL directa:
img: 'https://drive.google.com/uc?id=XXXXX',
09 — MINIATURAS NOTICIAS

Miniaturas del Blog

Cada noticia tiene una imagen de portada (miniatura) que aparece en la tarjeta del blog.

📸
Especificación
Tamaño: 800 × 450 px (proporción 16:9). Formato: JPG. Peso máximo: 150 KB.

El campo a completar en el array NEWS es:

img: 'img/noticias/aceite-diesel.jpg',

Si no ponés imagen (img: ''), se muestra el emoji de la noticia centrado, que también se ve bien.

10 — FOTO DEL TALLER

Foto Sección "Sobre Nosotros"

📸
Especificación
Archivo: img/taller.jpg. Tamaño: 800 × 600 px. Puede ser horizontal o cuadrada. Sugerencia: foto del frente del taller, del equipo, o del interior.

Una vez que tenés la foto guardada en img/taller.jpg, en index.html buscás el comentario nosotros-photo y reemplazás el emoji por la imagen:

12 — NOTICIAS

Agregar Noticias al Blog

Las noticias se gestionan en el array NEWS dentro de index.html. Buscás (Ctrl+F) el texto const NEWS = [.

Estructura de una noticia:

{
  id: 4,                                          // Número único
  title: 'Título del artículo',                   // Título visible
  excerpt: 'Descripción breve del contenido...',  // Texto corto (2-3 líneas)
  cat: 'Mantenimiento',                           // Categoría (texto libre)
  date: 'Junio 2025',                             // Fecha visible
  time: '4 min',                                  // Tiempo de lectura
  emoji: '🔧',                                    // Emoji si no hay foto
  img: 'img/noticias/mi-articulo.jpg',            // Miniatura (opcional)
  link: '#',                                      // URL del artículo completo
},
📝
Artículos completos
Por ahora el campo link apunta a # (la misma página). Cuando quieras publicar artículos completos, podés crear un archivo blog/nombre-articulo.html y apuntar ahí: link: 'blog/aceite-diesel.html'
13 — FLUIDDB

Agregar Vehículos al FluidDB

El módulo FluidDB está en fluidos.html. Para agregar un vehículo nuevo, necesitás los datos del manual del propietario (o del sitio toyota-club.net) y agregarlo al array DB.

🔗
Flujo de trabajo recomendado
Mandame el link de toyota-club.net del vehículo que querés agregar (como venimos haciendo en este chat) y yo extraigo los datos y te genero el bloque JSON listo para copiar y pegar.

La estructura de cada vehículo en el array DB dentro de fluidos.html:

{
  id: "camry_80",           // ID único (sin espacios)
  brand: "Toyota",          // Marca
  model: "Camry",           // Modelo
  generation: "80",         // Generación o código
  years: "2024-",           // Años
  market: "us",             // jp | us | eu | au | cn | gen
  variants: [
    { code: "ASV80", engine: "A25A-FKS", displacement: "2.5i",
      power_kw: 152, drive: "2WD", trans: "8AT" },
  ],
  fluids: {
    engine_oil: [
      { spec: "A25A-FKS", capacity_l: 5.0, type: "0W-20 API SN" }
    ],
    // ... demás sistemas
  }
},

16 — DOMINIO

Comprar el Dominio en Netlify

1
Crear cuenta gratuita
Entrás a netlify.com → Sign up. Podés usar Gmail. No requiere tarjeta para empezar.
2
Subir el sitio
Add new site → Deploy manually. Arrastrás la carpeta torke-site completa al área de upload. En 30 segundos el sitio queda en una URL temporal.
3
Agregar dominio
En el panel del sitio → Domain management → Add a domain → Buy a new domain. Escribís torke.com, si está disponible lo comprás. Costo: ~$15 USD/año con tarjeta.
4
HTTPS automático
Netlify configura automáticamente el certificado SSL. torke.com queda con HTTPS activo en 5-10 minutos, sin hacer nada más.
17 — ACTUALIZACIONES

Actualizar el Sitio

Cada vez que modificás algo (nuevo producto, nueva foto, texto editado), seguís este proceso:

1
Editás los archivos localmente
Modificás index.html o fluidos.html con Notepad++. Copiás las fotos nuevas a sus carpetas correspondientes.
2
Abrís Netlify
Entrás a app.netlify.com → tu sitio → pestaña Deploys.
3
Subís la carpeta actualizada
En la zona de drag & drop de Deploys, arrastrás la carpeta torke-site completa. Netlify detecta los cambios y actualiza el sitio en segundos.
⚠️
Siempre subí la carpeta completa
No subas solo el archivo que modificaste — subí siempre la carpeta torke-site completa, con todos los archivos e imágenes. Si subís solo el HTML, las imágenes desaparecen del sitio.

¿Cometiste un error?

Netlify guarda el historial de todos los deploys anteriores. En la pestaña Deploys podés hacer clic en cualquier deploy anterior y elegir "Publish deploy" para volver a esa versión instantáneamente.
